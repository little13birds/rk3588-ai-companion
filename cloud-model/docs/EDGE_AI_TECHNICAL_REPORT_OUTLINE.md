本文档用于规划参赛技术报告，面向“选题1：边缘人工智能应用”。报告内容以板端真实项目为准：

## 1. 作品定位

建议作品名称：

> 基于 RK3588 的边缘智能儿童陪护与安全守护机器人

核心定位：

本作品以 RK3588 为边缘 AI 主控平台，结合 RGB-D 相机、语音交互、端侧多模型推理、ROS2 机器人控制和家长端网页，实现面向家庭儿童场景的智能陪护与安全守护系统。系统重点体现边缘人工智能应用能力：多模型端侧部署、实时视觉感知、语音大模型工具调用、多模式资源调度，以及从感知到执行再到记录的闭环。

## 2. 摘要写作要点

摘要控制在 800 字以内，建议覆盖：

- 家庭儿童陪护中的实际问题：摔倒、危险物品接近、儿童独处、家长远程查看、阅读陪伴。
- 平台：RK3588 / ELF 2，运行嵌入式 Linux 与 ROS2。
- 边缘 AI：姿态、手部、危险物品、人脸识别、人体跟踪等模型在板端运行。
- 智能交互：KWS/VAD/ASR、TTS、大模型 Function Calling。
- 机器人能力：找人、跟随、避障、读书机械臂辅助拍摄。
- 家长端：网页状态查看、人物库、事件记录、机器人控制。
- 明确说明系统已实现边缘 AI 闭环，未实现内容不写入摘要成果。

## 3. 第一部分：作品概述

### 3.1 功能与特性

已实现或已集成的功能：

- 语音唤醒、语音识别、自然语言对话。
- 故事模式多音色朗读。
- 摔倒检测与安全提醒。
- 手部接近刀、剪刀、叉子等危险物品检测。
- 人脸录入、查询、删除与身份识别。
- 指定人物寻找、跟随与到达反馈。
- RGB-D 深度相机避障。
- 读书模式：机械臂/读书相机辅助对齐书本、拍摄书页、书本数据库匹配、朗读匹配内容。
- 家长端 Dashboard：实时画面、看护状态、人物管理、睡眠孩子设置、机器人控制、事件记录。
- SQLite 事件/对话/活动记录框架。
- 平台相机与读书相机动态切换。

待实现或规划中的功能，不能写成已完成：

- 书本内容多音色朗读。
- 大模型解释书本内容、提问和扩展讲解。
- 无书本匹配时的自由点读。
- 下一页没有扫描到时基于上下文继续阅读。
- ESP32 辅助拍摄图书内容。
- 手机端家长消息推送。

### 3.2 应用领域

- 家庭儿童安全陪护。
- 儿童阅读与故事陪伴。
- 家庭服务机器人。
- 智能家居边缘 AI 感知终端。
- 面向弱照护场景的人机交互与安全监测。

### 3.3 主要技术特点

- RK3588 NPU 部署多个 RKNN 模型，减少对云端视觉推理的依赖。
- ROS2 统一连接深度相机、底盘、机械臂和运动控制节点。
- `cloud-model` 主控工程作为调度层（当前板端 worktree 路径为 `/home/elf/cloud-model-safety-mainline`），协调语音、视觉、安全、读书、人物任务和 Dashboard。
- 安全守护采用端侧检测 + 事件去抖 + TTS 提醒 + Dashboard 记录的闭环。
- 人物任务采用人脸身份与人体检测/跟踪结合，实现指定目标寻找和跟随。
- 读书模式采用机械臂对齐、书页拍摄和书本数据库匹配。
- 运行时调度器管理 normal / reading / person task 等模式，降低 USB、相机、NPU、TTS、机械臂资源冲突。

## 4. 第二部分：系统组成及功能说明

### 4.2 硬件系统介绍

建议分模块写：

- RK3588 / ELF 2 主控板：边缘 AI 推理和主程序运行平台。
- Orbbec RGB-D 相机：平台视觉、深度避障、人物寻找与跟随、安全监控输入。
- 外置 USB 麦克风扬声器：语音唤醒、ASR 输入、TTS 播放。
- 移动底盘：执行找人、跟随和家长端控制。
- 读书机械臂：辅助书本拍摄和对齐。
- USB 拓扑：说明平台相机、音频、机械臂串口、底盘等外设同时存在，系统需要资源调度和健康检查。

## 6. 第四部分：典型工作流程

### 6.1 普通语音对话

```text
KWS 唤醒 -> VAD/ASR -> 意图判断 -> LLM Function Calling -> TTS 播报 -> Dashboard 记录
```

### 6.2 安全报警

```text
RGB 图像 -> 姿态/手部/危险品模型 -> 风险判断 -> 事件记录 -> TTS 提醒 -> 家长端查看
```

### 6.3 读书模式

```text
语音进入读书模式
  -> runtime_scheduler 切换资源
  -> 暂停或降低安全守护占用
  -> 启动读书机械臂服务
  -> 对齐书本
  -> 拍摄书页
  -> 数据库匹配
  -> 匹配成功后朗读
  -> 下一页或退出
```

注意：报告中不能写“匹配失败时大模型解释书本内容”作为已实现能力，只能写为规划方向。

### 6.4 找人和跟随

```text
语音/网页发起目标
  -> 解析 nearest / 角色 A/B / unique_name
  -> 查询人脸身份库
  -> 启动 person_seek/person_follow
  -> 旋转搜索和人体检测
  -> 人脸身份绑定人体 ID
  -> 深度测距和避障
  -> 到达或持续跟随
```

### 6.5 家长端管理

```text
Dashboard 请求
  -> cloud-model HTTP API
  -> SQLite/运行状态/相机 snapshot
  -> 页面展示和控制
```

## 8. 第六部分：关键技术难点与解决方案

建议写这些难点：

- 多个 RKNN 模型同时存在时的 NPU/CPU 资源协调。
- 平台相机、读书相机、USB 音频、机械臂串口共享 USB 拓扑时的冲突。
- Orbbec 热插拔后进程残留但 ROS graph 无节点的问题。
- 读书模式和安全守护之间的资源切换。
- ASR 长句、打断、TTS 播放之间的状态机问题。
- 找人任务的状态缓存和网页显示一致性问题。
- 安全检测中高召回与误报之间的权衡。
